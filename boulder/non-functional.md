## Scalability

- **Architecture:** Boulder is a decomposed, multi-component system. Each logical concern (CA, RA, VA, SA, OCSP generator, CRL generator/storer, Publisher, Nonce service, Akamai purger) is a separate gRPC service, allowing each to be scaled independently.
- **Statelessness:** gRPC service components are effectively stateless with respect to per-request data; durable state is delegated to MySQL (via SA) and Redis. This supports horizontal scaling of all non-storage components.
